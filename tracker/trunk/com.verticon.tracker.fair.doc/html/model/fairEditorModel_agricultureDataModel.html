Fair Data Model

Because all user activity centers around the creation and manipulation of Fair Documents, understanding the structure (called the data model) of Fair documents is the easiest way to learn to use the information management facilities of the Fair Editor.

Fair Data Model Core Elements

All Fair documents conform to a very simple structure called the Fair Data Model. In the Fair Data Model, a single Fair element is given a name and a reference to a Tracker Premises file. Within this Fair element Youth Clubs, People, and Divisions of Exhibits are created by the operator.

Core elements of the Fair Data Model are shown in Figure 5, “Fair Core Data Model Elements”.

Figure 5. Fair Core Data Model Elements


  1. The core elements of the Core Model are depicted as rectangles in the figure.

  2. The notation within the elements are the attributes of the element.

  3. The notation between the elements are the relationships of the elements.

The following table offers a text explanation of the Element Relationships.

Table 1. Core Elements of the Premises Model

Element NameRelationship to other Elements
FairThere is one Fair element. It acts as the first or root element of the document
name attributeA Fair has a name that is assigned by the operator
premises attributeA Fair has a premises attribute to reference the Premises document that will contain all the animal information
Person ElementsFair has one or more person elements that are used to document people contact information. A fair document can specify people as exhibitors, parents, supervisors and judges.
Youth Club ElementsExhibitors can be assigned to groups called Youth Clubs.
ExhibitsExhibits are assigned within a hierarchy of Divisions, Departments, Classes and Lot containers. See the Figure 6, “Hierarchy of Exhibits” for the diagram showing how the hierarchy contains multiple child elements.
