Tracker Data Model

Because all user activity centers around the creation and manipulation of Tracker Premises Documents , understanding the structure of Tracker Premises documents is the easiest way to learn to use the information management facilities of the Tracker.

All Tracker Premises documents structurally conform to the Tracker Premises Model . In the Tracker Premises Model a single Premises contains zero or more animals. Each Animal must have at least one but may have more than one (identification) tag. Each Tag must have at least one but may have more than one associated event.

In Figure 1, “Premises Core Model” below you will find a graphic representation of the core elements of the Tracker Core Premises Model.

Figure 1. Premises Core Model


  1. The core elements of the Premises Model are depicted as the blue rectangles in the figure.

  2. The notation within the elements are the attributes of the element.

  3. The notation between the elements are the relationships of the elements.

The following table offers a text explanation of the Element Relationships.

Table 1. Core Elements of the Premises Model

Element Name Relationship to other Elements
Premises There is one premises in a document
Animal A premises can have zero or many animals
Tag An Animal can have one or many tags
 A Premises can have zero or many tags that are unassigned
Event A Tag can have one or many events
