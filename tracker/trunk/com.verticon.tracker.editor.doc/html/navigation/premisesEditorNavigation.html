Navigating within Animal Documents

Tracker Model Editor

As seen in Figure 3, “Choosing a Premises Editor to Open a Premises Document” are two Tracker Editors used to manipulate Premises and Animal Documents, Tracker Model Editor and Tracker Transactional Model Editor.

Figure 3. Choosing a Premises Editor to Open a Premises Document


  1. The Tracker Model Editor is used for creating, manipulating, navigating and interpreting Premises and Animal Documents.

  2. The Tracker Transactional Model Editor is used to add online Event Transactions to Premises Documents.

Note

This chapter documents the navigation functionality of the Tracker Model Editor. The Tracker Model Editor is also used for bulk event processing. For more information on bulk even processing see the section called “Bulk Event Processing” .

Figure 4. Tracker Model Editor Pages Synchronized with the Outline View


The Tracker Model Editor is composed of several editor pages, Selection, Parent, List, Tree, Event History Table and Animals Table. Each page presents a different way of looking at the Elements in a Premises Document.

The Outline View is synchronized with each page of the Tracker Model Editor and the Properties View. By selecting an element in the Outline View, that Element is also selected in all Tracker Model Editor pages and it's attributes are visualized in the Properties View.

The Selection Page of the Tracker Model Editor is the main tree editor where all elements, their children, and attributes can be accessed. In the Editor toolbar area the Tracker Model Editor provides additional functionality for collapsing and expanding the tree in the Selection Page; synchronizing the Selection Page from the Animals and Events Pages; and for focusing all pages on the Selection Page's selected elements.

Note

The Tracker Model Editor also supports an advanced element selection and query functionality, for details see Tracker OCL Query Feature.
