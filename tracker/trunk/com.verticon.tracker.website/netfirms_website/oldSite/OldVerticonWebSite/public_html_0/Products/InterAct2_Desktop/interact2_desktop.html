The InterAct2 Desktop Integration Suite is the first product in a family of network management integration applications available from Verticon,  Inc. that is based on the technology we call InterAct2.

Verticon built InterAct2 as a data extraction and integration infrastructure.  The Desktop Integration Suite uses a InterAct2 integration engine within a Desktop application to provide analysts, consultants, and engineers with the tools they need to work with data from multiple sources. 

These individuals are routinely required to extract, integrate, and analyze data from their client's multiple network management applications.  Until now, the collection of client information was done by consultants on an ad hoc basis.   By automating this process, the InterAct2 frees these valuable individuals from performing tedious, time consuming, and error  prone tasks, allowing them to concentrate on other important work requiring  their skills.  Thus, the InterAct2 is a major step forward in improving  their productivity.

Leveraging state of the art software components and document driven integration techniques,  the InterAct2 is built on a dynamic document driven  runtime infrastructure.  This infrastructure enables the user to:

  • extract data from multiple data sources;
  • process it to a common format;
  • visualize it;
  • manipulate it;
  • persist it;
  • export or update it.

The dynamic runtime behavior of the InterAct2 is determined by integration specifications described in XML based integration documents.  These integration documents provide a very powerful, yet suprisingly easy way to specify (and implement!) integration activities.  Your custom integration specifications efforts are made even easier with the 50 example integration documents we have included with the tool.   

We are also building example and vendor specific integration 'Catalogs' for a variety of partner applications. Initially, we expect to support  HP/OpenView's Network Node Manager.


Much of the infrastucture of the InterAct2 is reusable.  Therefore Verticon will be devoting much of its development effort this year to building custom integration applications for Service Provider's.  For example, we could build an application for exporting managed elements from popular provisioning systems into reporting systems, performance data export from reporting systems into modeling and simulation engines, topology data import into physical inventory systems, and so on.

We look forward to working with network managers to define and implement applications which have broad applicability to popular network management applications.
