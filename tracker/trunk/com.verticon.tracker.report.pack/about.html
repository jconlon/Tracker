Introduction

The Tracker Report Pack is a suite of BIRT Reports and Resources that reference premises documents as data sources. Tracker Report Pack reports do not contain hard coded paths to premises documents, so they need no modification to target different premises documents.

On installation, Tracker Report Packs will be detected and its reports and resources will deployed to the workspaces ReportViewer Servlet Working Path folder (specified by the birt.viewer.root.path system property), Reports can then be called via a Browswer with URL parameters using relative non-system specific paths.

Datasources

Tracker Report Pack reports utilize property files or hidden report parameters to set up the datasource documents. If the report is run with the Report Parameters embedded in the url
&rp_premises_doc=platform%3A%2Fresource%2FExample_Fair%2Fexample.premises then the report will use the example.premises document from the Example_Fair project as the datasource. If the report parameters are not passed during launch then the report will load the local report.properties file and the value of the rp_premises_doc parameter.

Navigator Popup Menus

If the report parameter is not passed during launch as is the case for launching from the Navigator, then the report will attempt to load a report.properties file from the same folder as the report and from this proprties file attempt obtain the value of the rp_premises_doc parameter.

Shared Resources

To minimize duplication the Tracker Reports infrastructure utilize a shared BIRT resource library.

Installation

On boot up Tracker Report Pack Reports and Shared Resources are automatically installed in the working path directories of the Report Viewer.

  1. Find the default shared resource directory on your machine. It should be located in the Tracker installation

tracker/plugins/org.eclipse.birt.report.viewer_*/birt

  1. Create a resources folder in that directory
  2. Add the libraries in the Library directory of this project to the tracker/plugins/org.eclipse.birt.report.viewer_*/birt/resources folder
  3. Add any files in the Styles directory to the tracker/plugins/org.eclipse.birt.report.viewer_*/birt/resources folder
  4. Copy report(s) to tracker/plugins/org.eclipse.birt.report.viewer_*/birt/report folder

Optional

  1. Copy report(s) to target project
    1. Copy the report.properties file to the target project in the same directory as the copied reports
    2. Edit the report.properties to reference the target premises document.

Animal Events Report Plan

Purpose

What is the purpose of the report?

The purpose of this report is to show the events of all animals. Animal identification, types and a list of events will be shown.

Target

Who is going to read the report?

Livestock premises managers and workers.

Data

What data should appear in the report?

  1. ID
  2. Visual ID
  3. Species
  4. Sex
  5. ADG
  6. Age
  7. Event List
    1. Date of Event
    2. Event Type
    3. Comments

Source

Where is the data coming from?

Premises document, via the EMF DataSource.

Calculated Data

Does any data need to be calculated?

None

Calculation Methods

How will the data be calculated?

N/A

Interaction

Do you want to enable the report user to specify what data to display?

Not at this time, but it in the future we may want to reuse this report, sharing it across projects.

Animal Locations Report Plan

Purpose

What is the purpose of the report?

The purpose of this report is to show the last known location of all animals. Animal identification, types, weights, and locations will be shown.

Target

Who is going to read the report?

Livestock premises managers and workers.

Data

What data should appear in the report?

  1. ID
  2. Visual ID
  3. Species
  4. Breed
  5. Sex
  6. Weight
  7. ADG
  8. Age
  9. Date of last sighting
  10. Location

Source

Where is the data coming from?

Premises document, via the EMF DataSource.

Calculated Data

Does any data need to be calculated?

Yes, for highlighting animals with ADG under 3 lbs.

Calculation Methods

How will the data be calculated?

In a highlighting trigger on the ADG column of the report.

Interaction

Do you want to enable the report user to specify what data to display?

Not at this time.
